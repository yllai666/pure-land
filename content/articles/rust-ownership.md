## 所有权规则

Rust 的所有权系统有三条核心规则：

1. Rust 中每一个值都有一个**所有者**
2. 同一时间只能有一个所有者
3. 当所有者离开作用域，值将被释放

这三条规则看起来简单，但衍生出的概念——借用、生命周期、移动语义——构成了 Rust 最陡峭的学习曲线。

## 借用

与其转移所有权，我们更常用的方式是**借用**：通过引用让函数临时使用值而不获取所有权。

```rust
fn main() {
    let s = String::from("hello");
    print_str(&s);  // 借用 s，所有权不变
    println!("{}", s);  // 仍然可用
}

fn print_str(s: &String) {
    println!("{}", s);
}
```

## 可变引用

可变引用有且仅有一个的限制，这是 Rust 在编译期防止数据竞争的机制。一开始觉得很不自由，但理解之后会发现它把很多运行时 Bug 提前到了编译期。

Rust 的学习曲线是真的陡，但一旦越过所有权系统这道坎，后面的路会顺畅很多。
